‍How does it work? VRAM is organized in 16-bit words. VMAIN = 0x80 configures auto-increment on the high byte write (VMDATAH). Each pair of writes fills one VRAM word.

Tips & Tricks

Practical tips and common pitfalls specific to this example:

  • VRAM and VBlank: VRAM writes are silently ignored during active display. If your display is corrupted, make sure you write during WaitForVBlank().
  • Palette color 0: always the backdrop. If your screen is black but tiles are loaded, check that color 0 isn't set to black.

Go Further

Concrete modifications to try on this example:

  • Change the colors: modify the REG_CGDATA values and see what happens. Format is BGR555 (5 bits per channel, little-endian).
  • Add animation: in the main loop, increment a variable each frame and use it to change the scroll offset or palette.
