Detailed Description

SNES Sprite (Object) Management.

Functions for managing hardware sprites (OBJ).

SNES Sprite Limits

  • 128 sprites total
  • 32 sprites per scanline
  • Sizes: 8x8, 16x16, 32x32, 64x64
  • 8 palettes (16 colors each, sharing color 0)

Usage

// Initialize sprite system
oamInit();
// Set up a sprite
oamSet(0, 100, 80, 0, 0, 0, 0); // Sprite 0 at (100, 80)
// In main loop
WaitForVBlank();
oamUpdate(); // Copy OAM buffer to hardware

X-high-bit mask for a sprite slot (0-3 within a high-table byte)

Each high-table byte covers 4 sprites. Bit 0 of each 2-bit pair is the X high bit. This macro returns the mask for the X-high bit of the given slot.

◆ oamInitDynamicSpriteEndFrame()

void oamInitDynamicSpriteEndFrame ( void  )

End frame processing for dynamic sprites.

Call after drawing all sprites each frame. Hides sprites that were visible last frame but not drawn this frame, and resets per-frame counters.

// Game loop
while (1) {
for (i = 0; i < numSprites; i++) {
oamDynamic16Draw(i);
}
oamInitDynamicSpriteEndFrame(); // Hide unused sprites
WaitForVBlank();
oamVramQueueUpdate(); // Upload graphics to VRAM

void oamVramQueueUpdate ( void  )

Process VRAM upload queue.

Call during VBlank to upload queued sprite graphics to VRAM. Transfers up to 7 sprites per frame to stay within VBlank budget. If more sprites are queued, they will be transferred on subsequent frames.

Note
Must be called after WaitForVBlank() for proper timing.
